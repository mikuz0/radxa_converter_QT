# Radxa ROCK 4D - Удаленный перекодировщик видео

Версия 3.1.0

## 📋 Описание

Программа для удаленного управления аппаратным перекодированием видео на одноплатном компьютере Radxa ROCK 4D. Поддерживает подключение по SSH, аппаратное кодирование через RKMPP, пакетную обработку и многое другое.

## ✨ Возможности

### 🚀 Основные функции
- **Подключение по SSH** к Radxa с сохранением настроек
- **Файловый менеджер** с полной навигацией по удаленной файловой системе
- **Загрузка/скачивание файлов** с отображением прогресса
- **Создание/удаление папок и файлов** на удаленной машине

### 🎬 Кодирование видео
- **Аппаратное кодирование** через RKMPP (hevc_rkmpp, h264_rkmpp)
- **Программное кодирование** (libx265, libx264)
- **Режим "только аудио"** для извлечения звуковой дорожки
- **Настраиваемые параметры**: разрешение, битрейт, аудиокодек, частота, каналы
- **Отображение прогресса** в реальном времени
- **Оценка размера** выходного файла с проверкой свободного места

### 📦 Пакетная обработка
- **Очередь файлов** для последовательной обработки
- **Сохранение очереди** между сессиями
- **Статусы файлов**: queued, processing, done, failed
- **Общий прогресс** по всей очереди
- **Автоматический переход** к следующему файлу

### 💾 Профили и настройки
- **Сохранение профилей** с параметрами кодирования
- **Загрузка профилей** из файла
- **Экспорт/импорт** профилей в JSON
- **Автосохранение** последних настроек

### 🖥️ Интерфейс
- **Drag & drop** файлов
- **Контекстное меню** в файловом менеджере
- **Фильтрация файлов** по имени
- **Лог выполнения** с временными метками (скрывается по умолчанию)
- **Статусная строка** с информацией о соединении
- **Горячие клавиши**:
  - `F1` / `Ctrl+H` - справка
  - `Ctrl+O` - открыть файловый менеджер
  - `Ctrl+S` - старт конвертации
  - `Ctrl+Q` - выход

## 🔧 Системные требования

### Для сборки:
- Ubuntu 22.04 или новее (включая Ubuntu 25.10)
- Qt5 (>= 5.12)
- libssh (>= 0.8)
- CMake (>= 3.16)
- GCC (>= 9.0)

### Для работы:
- Radxa ROCK 4D с установленным ffmpeg (с поддержкой RKMPP)
https://mkuznecov.ru/linux/kompiljacija-i-ustanovka-ffmpeg-s-podderzhkoj-rockchip/
- SSH доступ к Radxa

## 📦 Установка зависимостей

```bash
# Ubuntu 22.04 / 24.04 / 25.10
sudo apt update
sudo apt install qtbase5-dev qtbase5-dev-tools qtchooser qt5-qmake \
                 libqt5core5t64 libqt5gui5t64 libqt5widgets5t64 \
                 libqt5network5t64 libqt5concurrent5t64 \
                 qttools5-dev-tools cmake build-essential \
                 libssh-dev pkg-config
                 
                 
                 # Клонирование репозитория (или создание структуры проекта)
git clone <url-репозитория> radxa-converter
cd radxa-converter

# Сборка
mkdir build && cd build
cmake ..
make -j$(nproc)

# Запуск
./radxa_converter

Использование
1. Подключение к Radxa

    Введите IP адрес вашего Radxa (по умолчанию 192.168.1.135)

    Введите логин (обычно rock или root)

    Введите пароль

    Нажмите "Подключиться"

2. Выбор файла

    Откройте файловый менеджер кнопкой "📁 Файловый менеджер"

    Перейдите к нужному файлу

    Двойной клик для выбора файла

    Или используйте Drag & Drop

3. Настройка параметров

    Выберите кодек (аппаратный или программный)

    Укажите разрешение (или оставьте пустым для оригинального)

    Настройте битрейт видео

    Выберите аудиокодек и параметры

    При необходимости включите режим "только аудио"

4. Оценка размера

    Нажмите "Оценить размер" для примерного расчета

    Программа покажет ожидаемый размер и проверит свободное место

5. Запуск конвертации

    Нажмите "▶ Начать перекодирование"

    Следите за прогрессом в реальном времени

    По окончании можно скачать результат

6. Пакетная обработка

    В файловом менеджере выберите несколько файлов

    Нажмите "Добавить в очередь"

    Откройте окно очереди кнопкой "📋 Очередь"

    Управляйте порядком файлов

    Нажмите "▶ Запустить" для начала пакетной обработки

⚙️ Параметры кодирования
Видео кодеки:

    hevc_rkmpp - аппаратное H.265 (рекомендуется для Radxa)

    h264_rkmpp - аппаратное H.264

    libx265 - программное H.265

    libx264 - программное H.264

    copy - копирование без перекодирования

Аудио кодеки:

    copy - копирование без перекодирования

    aac - хорошее качество, малый размер

    mp3 - стандартный формат

    ac3 - формат для DVD/Blu-ray (5.1)

    flac - без потерь

    opus - современный формат

Каналы:

    1 - моно

    2 - стерео

    6 - 5.1 surround

Шаблон имени:

    {name} - исходное имя файла

    {codec} - выбранный кодек

    {res} - разрешение

📝 Примечания

    Аппаратное кодирование требует прав доступа к /dev/dri/

    При ошибках аппаратного кодирования рекомендуется использовать программные кодеки

    Пароль хранится в зашифрованном виде в ~/.config/Radxa/RadxaConverter.conf

    Очередь сохраняется между сессиями и автоматически загружается при запуске

🐛 Известные проблемы и решения
Проблема	Решение
"Failed to init MPP context"	Используйте программное кодирование (libx265)
Файловый менеджер не открывается	Проверьте подключение к Radxa
Прогресс не отображается	Убедитесь, что ffmpeg установлен на Radxa
